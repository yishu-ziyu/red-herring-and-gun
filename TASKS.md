# TASKS：红鲱鱼与枪产品对齐清单

## 使用方式

这个文件不是开发任务清单，而是产品对齐清单。

当前开发任务和验证历史以 `mvp/tasks/todo.md`、`mvp/DEVELOPMENT_LOG.md` 和 Git 历史为准。

本文件早期内容来自「语境化可核查分解 Agent」规划阶段。项目现在已经进入「红鲱鱼与枪 / 信息真相猎人」可运行 MVP 后期，所以这里保留产品问题，但不再表达“尚未进入实现”。

## 当前讨论原则

- 先对齐真实核查路径，再改 UI 或 Agent 编排。
- 不一次性塞太多产品框架。
- 每轮聚焦一个可验证问题。
- 边聊边保存。
- 方法论已经沉淀在 `HANDOFF.md` 和种子规则文档中；当前重点是让实现、路演叙事和证据闭环一致。

## 当前优先对齐问题

### A. 黑客松主叙事

当前采用「信息真相猎人」外壳：

> 先别转发，先看证据链。

需要持续检查：页面、演示和报告是否都围绕来源追踪、交叉验证、逻辑审计、证据链呈现，而不是泛泛展示“多 Agent 很多”。

### B. MVP 闭环

当前实现已经有文字、链接、图片材料入口和 Mission Control 办案台。后续对齐重点：

1. 失败的搜索或模型调用要在证据链里可见，不能被静默抹平。
2. 最终报告必须说明 can say / cannot say。
3. 闭环动作要具体，例如生成可转发核查卡、记录记忆候选、标记未解问题。
4. 页面里的每个 Agent 模块都要绑定真实状态，不能只做装饰。

### C. Demo 优先用户

黑客松演示优先面向“看到热点传言、截图、链接后需要快速判断能不能转发的人”。

本科生、研究生、记者、内容创作者、政策研究者仍是长期扩展用户，但不是当前演示的唯一中心。

### D. 当前不再优先的问题

以下早期问题保留为历史脉络，不再阻塞当前实现：

- 是否进入 MVP。
- 是否先只做方法论讨论。
- MVP 是否只做“路径卡片 + 右侧报告”。
- 是否从零选择第一版用户。

## 历史讨论问题

### 1. 产品第一性目标

状态：已确认。

问题：这个产品优先解决的是写作质量，还是思考质量？

结论：优先解决思考质量，即问题拆解、证据判断、概念澄清、因果识别。

写作质量只是最后的输出层，不是根问题。

---

### 2. MVP 首要用户

状态：已被黑客松叙事收敛。

问题：虽然本科生、研究生、记者、内容创作者、政策研究者、知识工作者都可能使用，但 MVP 第一版最应该为谁优化？

当前结论：黑客松 Demo 优先服务热点传言、截图、链接核查场景，使用“信息真相猎人”叙事；学术和政策场景保留为长期版本。

---

### 3. 核心用户动作

状态：已被当前实现收敛。

问题：用户在产品里最重要的动作是什么？

当前结论：核心动作是添加待核查材料并启动真实核查；之后由主控办案台展示拆解、溯源、交叉验证和闭环。

---

### 4. MVP 闭环

状态：已进入实现验证。

问题：两到三天内的 MVP 最小闭环是什么？

当前结论：必须闭环。当前实现走“输入材料 -> Mission Control -> 流式 Agent 编排 -> 多引擎搜索 -> 证据交叉验证 -> 报告/闭环”的路线。

---

### 5. 白板式交互的 MVP 形态

状态：已被 Mission Control 办案台替代。

问题：是否做真正无限白板，还是先做路径卡片？

当前结论：长期仍可保留可增长图谱愿景；当前 MVP 以办案台和证据链工作流为主，不再把“路径卡片 + 右侧报告”作为唯一形态。

---

### 6. 证据等级标准

状态：仍需随实现校准。

问题：如何定义证据强弱？

候选等级：

- A：原始权威来源
- B：专业二手分析
- C：解释性材料
- D：个案与舆论线索
- E：不可用或高度可疑

待校准：当前证据等级要和 `mvp/src/lib/evidenceQuality.ts`、`mvp/src/lib/evidenceConsensus.ts`、`mvp/src/lib/sourceIndependence.ts` 的实现保持一致。

---

### 7. 不同用户身份的证据路径

状态：长期问题，当前非阻塞。

问题：用户身份如何影响证据需求？

候选路径：

- 学术研究路径
- 课堂理解路径
- 内容创作路径
- 新闻核查路径
- 政策分析路径

待讨论：是否让用户先选身份，还是 Agent 自动判断后建议。

---

### 8. 好输出的评估标准

状态：仍有效。

问题：什么样的拆分算好？什么样的报告算好？

当前标准：

1. 保留争议重点。
2. 不粗暴关键词化。
3. 单一判断关系。
4. 对应证据类型。
5. 可回到原论证。
6. 区分事实与价值。
7. 指出概念偷换、因果跳跃、比较对象缺失。

---

### 9. 和现有产品的差异

状态：仍有效。

问题：如何避免像 Perplexity、普通 Deep Research、RAG、搜索总结工具？

当前差异：

- 它们以信息为中心。
- 我们以论证结构和证据许可为中心。

---
