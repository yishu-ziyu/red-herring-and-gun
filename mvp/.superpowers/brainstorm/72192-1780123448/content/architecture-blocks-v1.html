产品能力板块地图

先把“规范”转成产品中真正可见、可解释、可演示的能力板块。点击一个选项不是最终决定，只是帮助我们记录讨论偏好。

推荐产品叙事：用户不是在看评分，而是在看一条调查路径
Block 1

可核查结论

Agent 如何从原始 claim 收束到谨慎结论。

已具备：MissionControl + ReportComposer

Block 2

溯源与交叉验证

多搜索源、原始来源、反证、转载同源识别。

半完成：360 双向检索，矩阵待接真实数据

Block 3

场景覆盖

健康、社会、科技、财经、政治、娱乐等类型。

已具备：Demo 类型和 RumorDetector 模式识别

Block 4

结果闭环

辟谣卡片、存疑归档、分享、导出。

已具备：ConclusionDock

Block 5

Agent 记忆/知识库

历史案例、历史证据、相似 claim 复用。

已具备：localStorage 版 Memory

Block 6

国产模型与 360 联动

StepFun、360 智脑、360 AI Search 作为底层能力。

已具备：模型链路和 360 搜索链路

第一轮要锁定的不是 UI 细节，而是主叙事结构

A

六板块能力地图

把产品讲成六个可见能力：结论、溯源、覆盖、闭环、记忆、模型生态。后续逐块 grill-me。

B

三阶段用户流程

只讲输入态、执行态、结果态。更简洁，但容易把知识库、闭环和生态能力藏掉。

C

证据矩阵为中心

把所有能力都围绕证据矩阵展开。表达锋利，但可能让产品显得过窄。
