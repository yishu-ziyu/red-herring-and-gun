# 溯证 Agent — 黑客松 Demo 演示脚本

> **目标**：3 分钟 demo + 1 分钟 elevator pitch
> **核心信息**：搜索引擎帮你找信息，溯证帮你看清一个观点的结构

---

## 一、Elevator Pitch（1 分钟版本）

> "你有没有过这样的经历：写论文时引用了一个看起来靠谱的观点，后来被导师问'这个证据能支持这个结论吗'，答不上来？
>
> 我们的溯证 Agent 做一件事：你把一句论断扔给它，它自动拆解这句话里藏了几个判断、每个判断需要什么证据、你手里的材料能支持到什么程度——而不是直接告诉你'对'或'错'。
>
> 举个例子，'AI 导致初级内容岗位减少'这句话，听起来像事实，但它其实混合了概念定义、数量事实、机制判断和因果归因。我们的工具把它拆开，告诉你哪一步证据不足、不能说'导致'，最后帮你改写成更谨慎但同样有力的表达。"

---

## 二、3 分钟 Demo 脚本

### 阶段 0：开场钩子（0:00-0:15）

**屏幕**：黑屏，只显示一句话

> "AI 导致初级内容岗位减少。"

**话术**：
> "这句话看起来像一个事实陈述。但如果你要把它写进论文、研报或者新闻稿，有一个问题：它真的能被现有证据支持吗？"

**动作**：点击"进入语境化分析"

---

### 阶段 1：诊断与拆解（0:15-0:45）

**屏幕**：诊断横幅展开

**话术**：
> "我们的 Agent 第一步不是去搜索，而是先诊断这句话的结构。它发现这不是一个单一事实，而是混合了五种判断——概念、数量、机制、因果，还有反证路径。"

**动作**：指向诊断横幅中的"混合判断"标签

> "最关键的是这个'因果判断'。原句说'导致'，这是最危险的部分。"

---

### 阶段 2：进入画布，展示论证地图（0:45-1:15）

**屏幕**：Reasoning Canvas 显示全部节点

**话术**：
> "这是我们的核心界面——语境画布。中心是原句，第一层是五个判断节点。注意看：因果判断被标成了红色，因为 Agent 判断这里风险最高。"

**动作**：
1. 点击"因果判断"节点
2. Inspector 面板弹出，显示"能不能说'导致'？"

> "点击任意节点，Inspector 会告诉你这个节点在问什么、需要什么证据、不能用什么证据。"

---

### 阶段 3：聚焦模式 + 证据需求（1:15-1:50）

**动作**：双击"因果判断"节点，进入聚焦模式

**屏幕**：高亮因果判断 → 证据需求 → 候选证据的路径，其他节点淡化

**话术**：
> "双击进入聚焦模式，只看这条推理链。要支持'导致'，我需要四个东西：时间顺序、机制链、替代解释，还有反事实证据。"

**动作**：点击"时间顺序"证据需求节点

> "点击证据需求，Inspector 告诉我：AI 采用必须早于岗位变化。没有这个，不能谈因果。"

---

### 阶段 4：候选证据审计（1:50-2:20）

**动作**：退出聚焦，点击"招聘时间序列"候选证据节点

**屏幕**：Inspector 显示材料详情

**话术**：
> "现在我有一份材料——某招聘平台的数据显示内容写作类岗位在下降。Agent 告诉我：这份材料能支持'岗位需求下降'，但不能支持'AI 导致'，因为它缺少 AI 采用变量。"

**动作**：指向 Inspector 中的"可以说"和"不能说"区块

> "这就是我们的核心能力：不是告诉你材料对不对，而是告诉你它能支持什么、不能支持什么。"

---

### 阶段 5：Agent 展开——动态发散（2:20-2:50）

**动作**：点击"因果判断"节点，选择"寻找反证"，点击"调用中控 LLM"

**屏幕**：等待动画 → 新增 Agent 任务节点 → 新增反证结果节点

**话术**：
> "如果我觉得证据还不够，可以在任意节点上追问。Agent 会动态生成新的分支。比如我选择'寻找反证'，它会在画布上接入一个反向分支。"

> "注意：Agent 不会自动帮你走完所有路径。你在某个节点上追问，它才调度对应能力。这是故意的——我们要的是'人在回路'，不是黑箱替代。"

---

### 阶段 6：结论与改写（2:50-3:10）

**动作**：点击"推理许可"节点

**屏幕**：显示"证据不足，不能说'导致'"

**话术**：
> "最后，Agent 给出推理许可：现有证据最多支持'AI 可能是影响因素之一'，不能支持'导致'。"

**动作**：点击"降强度改写"节点

> "同时给出三个版本的改写——谨慎版、通俗版、研究备忘录版。你不是不能说 AI 的影响，而是要说到证据允许的程度。"

---

### 阶段 7：收尾（3:10-3:20）

**屏幕**：回到完整画布，Conclusion Dock 显示强度对比

**话术**：
> "从'导致'到'可能是影响因素之一'——这不是削弱你的观点，而是让你的观点站得住。搜索引擎帮你找信息，溯证帮你看清一个观点的结构。"

## 三、对照 Demo 设计

### 对照方案 A：搜索页面对比（推荐）

**实现方式**：在 DiagnosisBanner 页面添加一个"先看看传统搜索会给出什么"的折叠区块

**内容设计**：

```
┌─────────────────────────────────────────────────────────────┐
│  传统搜索 / ChatGPT 的回应                                   │
├─────────────────────────────────────────────────────────────┤
│  "是的，AI 确实在替代初级内容岗位。研究显示写作职业对大    │
│   语言模型暴露度较高，某招聘平台数据显示相关岗位下降，      │
│   多个企业案例证实 AI 减少了文案外包需求..."                │
│                                                             │
│  ⚠️ 问题：把'暴露度'、'岗位下降'、'企业案例'混为一谈，      │
│     直接推出'AI 导致'。每份材料的证据强度没有被区分。       │
└─────────────────────────────────────────────────────────────┘
                      ↓ 对比 ↓
┌─────────────────────────────────────────────────────────────┐
│  溯证 Agent 的回应                                           │
├─────────────────────────────────────────────────────────────┤
│  • 暴露度研究 → 能支持：任务结构可能变化                     │
│                   不能支持：岗位已经减少                     │
│  • 招聘数据 → 能支持：某口径下岗位下降                       │
│               不能支持：下降原因（缺少 AI 采用变量）         │
│  • 企业案例 → 能支持：单个企业流程变化                       │
│               不能支持：行业趋势                             │
│                                                             │
│  结论：现有证据不能支持"导致"，建议改写为                    │
│  "AI 可能是影响因素之一"                                    │
└─────────────────────────────────────────────────────────────┘
```

### 对照方案 B：前后对比（简洁版）

在 Conclusion Dock 中增加一个"如果不用溯证"的对比标签页：

| 维度 | 传统方式 | 溯证 Agent |
|------|---------|-----------|
| 输入 | 关键词搜索 | 完整论断 |
| 输出 | 材料列表 | 论证结构 + 材料分级 |
| 判断 | 对/错/不清楚 | 能支持什么、不能支持什么 |
| 改写 | 无 | 三版本强度调整 |
| 可追溯 | 低 | 每步有节点记录 |

## 四、演示节奏建议

### 快速版（2 分钟）

跳过阶段 3（聚焦模式细节）和阶段 5（Agent 展开），重点展示：
1. 诊断（混合判断）
2. 画布（节点结构）
3. 证据审计（可以说/不能说）
4. 改写结论

### 完整版（5 分钟）

按完整脚本走，增加：
- 多展开一个节点（如"概念不明"）
- 展示评论系统（点击节点添加评论）
- 展示 Trace Stack（左侧推理步骤）

## 五、常见评委问题预判

**Q1：这和 Perplexity / Consensus 有什么区别？**
> "Perplexity 帮你搜索和总结，但不告诉你'这句话里的因果推断是否有证据支持'。溯证做的是论证结构检查，不是信息检索。"

**Q2：如果用户不懂逻辑学，能看懂吗？**
> "我们的节点用语都是自然语言——'能不能说导致'、'缺少什么证据'。不需要懂逻辑学，只需要有'写论文时被质疑过引用'的经历。"

**Q3：这会不会让写作变得过于保守？**
> "恰恰相反。我们反对的是'把弱证据写成强结论'。如果你的证据足够强，Agent 会告诉你'可以保留原表述'。目标是准确，不是保守。"

**Q4：后端用的是什么模型？**
> "目前用 OpenAI GPT-4o 做中控调度，各子任务可以配置不同模型。搜索层可以接 Serper / Google / 任意搜索引擎。"
