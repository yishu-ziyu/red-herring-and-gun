# 溯证 Agent 3 分钟 Demo 讲稿

## 0:00 - 0:25 开场

今天我们做的是“溯证 Agent”，一个 Agent Reasoning Canvas。

它不是帮你把资料一次性搜满，也不是替你自动跑完整个架构，而是先把观点展开成一张问题地图，再让你选择想继续追的节点。

很多观点听起来很有力量，比如：

> AI 导致初级内容岗位减少。

但问题是，这句话可能说太满了。

## 0:25 - 0:55 输入观点

用户进入页面后，只需要粘贴一句想判断的观点，然后点击“开始推理”。

这个产品不要求用户先懂什么是因果识别、证据路由或 source-claim fit。用户只需要带着一句话进来，Agent 会把它放到白板中心，并先搭出前三层问题空间。

## 0:55 - 1:20 第一个 Aha Moment：观点长成地图

第一步，Canvas 里出现中心观点：

> AI 导致初级内容岗位减少。

接着它向外长出几个判断层节点：

- 概念不明
- 数量事实
- 机制判断
- 因果判断
- 反证路径

这一步的价值是：用户马上意识到，原句不是一个可以直接搜索后判真假的简单事实，而是一组互相牵连的判断。

## 1:20 - 1:50 Agent 搭出问题空间后暂停

接下来，Agent 会先展开最高风险的“因果判断”，但它不会无限自动跑下去。

因为原句里最强的词是：

> 导致。

Canvas 会长出几类证据需求：

- 时间顺序：AI 采用是否早于岗位变化？
- 机制链：任务替代如何传导到招聘需求？
- 替代解释：是否可能是行业周期、降本或平台变化？
- 反事实：没有 AI 时是否仍会下降？

到这里，系统会停住。因为如果 AI 继续替用户自动展开所有分支，用户会失去方向。

真正的交互应该是：

> 我对哪一个节点感兴趣，我就在那个节点上继续发散。

## 1:50 - 2:25 节点触发 Agent

这是产品最核心的能力。

用户点击“替代解释”节点，右侧 Inspector 会出现一个节点追问区。

用户可以输入：

> 我想沿这个节点继续追问：还需要哪些证据？

然后选择要触发的能力：

- 证据审计
- 联网搜索
- 反证生成
- 局部改写

这时才调用真实中控 LLM。

中控 LLM 通过本地 API 代理接入真实模型：如果本地配置了 OpenAI key，就走 OpenAI Responses API；如果没有 key，就调用当前 active 的本地 Anthropic-compatible proxy；再不可用时，兜底调用 Codex CLI。

它不直接输出答案，而是只判断：这个节点应该派给哪个子 Agent。

比如用户选择“联网搜索”，画布上只会在“替代解释”附近新增：

- 中控 LLM 调度
- Searcher 子 Agent
- 新增候选证据

如果选择的是“联网搜索”，后端会显式启用模型侧搜索能力；如果选择“证据审计 / 反证生成 / 局部改写”，则只围绕当前节点做局部推理，不主动联网。

这一步让系统像人脑思考：从一个点向外发散，但发散方向由用户选择。

## 2:25 - 3:00 不急着给最终答案

底部 Conclusion Dock 不会急着汇报最终答案，而是提示：

> 正在局部发散，暂不收束总答案。

这很重要。

因为一个好的 Agent 产品不是替用户把所有东西都展开，而是在用户需要某种能力时，及时把那个能力接到当前节点上。

长期来看，每个节点都可以成为一个触发点：

- 搜索证据
- 审计材料
- 生成反证
- 改写表达
- 继续拆分子问题

这就是溯证 Agent 的价值：

> 它不是替你跑完思考，而是让你在论证地图上调度思考。
