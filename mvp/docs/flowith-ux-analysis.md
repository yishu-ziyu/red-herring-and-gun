# Flowith UX 深度分析 & 溯证 Agent 改进建议

## 一、Flowith 信息架构

```
flowith.io/blank (首页/仪表盘)
    ├── 左侧边栏 (永久可见)
    │   ├── Logo + 收起按钮
    │   ├── 主导航: 新建 Flow / 画布管理 / 知识花园 / 知识市集
    │   ├── 会员促销横幅
    │   ├── 项目文件夹树 (可展开/折叠)
    │   ├── 置顶内容区
    │   └── 用户头像 + 套餐信息
    │
    └── 主内容区
        ├── 顶部通知横幅
        ├── 大标题 "Flow with ideas" + 副标题
        ├── 中央输入框 (每个伟大的想法都始于一个念头...)
        ├── 模型选择器 (GPT 4.1 下拉)
        ├── 模式选择栏 (对话/图像/视频/幻灯片/网页/Neo Agent)
        └── 模板卡片网格 (各种创作模板)

project/{id} (项目页)
    ├── 项目标题 (可下拉切换)
    ├── 个性化设置区 (背景/语气/输出规则)
    ├── Flow 区域
    │   ├── 搜索 Flow 名称
    │   ├── Flow 卡片网格 (+ 新建 Flow / 已有 Flow)
    │   └── 排序/视图切换
    └── Composer 区域

flow/{id} (画布页)
    ├── 顶部导航栏
    │   ├── Logo | Flow 名称 (可编辑下拉) | 分享/导出 | 全屏 | 返回
    │   └── 右侧: 积分状态 | 创作台按钮
    ├── 左侧浮动工具栏 (圆角白底竖条)
    │   ├── 搜索
    │   ├── 设置/模型
    │   └── 画布/笔记切换
    ├── 主体: 无限画布 (浅灰背景 #f5f5f5)
    └── 底部中央悬浮输入栏
        ├── 默认态: 简洁输入框
        └── 展开态: 模型选择 + 模式按钮 + 输入框 + 发送
```

## 二、关键设计模式提取

### 模式 1: 分层入口 — 不让用户直接面对空白画布

Flowith 不会让用户一进入就直接面对空白画布。它有三层入口：
1. **首页** — 展示品牌、输入框、模板（降低心理门槛）
2. **项目页** — 组织管理 Flow，先选/创建一个 Flow
3. **画布页** — 真正的创作空间

**对比溯证 Agent**: 当前直接进入画布，且画布上预置了复杂节点。用户没有"准备期"。

### 模式 2: 左侧边栏是"功能完整"的导航系统

Flowith 的左侧边栏每个项都有实际功能：
- **新建 Flow** → 创建新画布
- **画布管理** → 查看所有画布列表
- **知识花园** → 知识库管理
- **知识市集** → 模板/社区内容

底部还有项目文件夹树（可展开）、置顶内容、用户信息。

**对比溯证 Agent**: 当前四个按钮（画布/Agent/知识库/设置）点击无任何反应，只是视觉装饰。

### 模式 3: 底部中央输入栏 — 画布的核心交互锚点

Flowith 画布的核心交互不在顶部，而在**底部中央的悬浮输入栏**：
- 默认是一个圆角白底长条，占位符文字引导输入
- 输入后展开为完整工具栏（模型选择 + 模式切换）
- 发送后，用户问题生成一个圆角白色节点在画布中央
- AI 回复生成新的节点，形成树状展开

**对比溯证 Agent**: 当前输入在顶部 header 区域，与画布分离。画布是"只读"的，用户不能直接在画布上发起新任务。

### 模式 4: 节点样式 — 极简卡片，内容优先

Flowith 节点设计：
- 圆角白色卡片，浅阴影
- 文字内容为主，无复杂边框/颜色编码
- 底部有来源标识（"我" / AI 模型图标）
- 时间戳右对齐
- 节点之间用细线连接

**对比溯证 Agent**: 当前节点是按钮样式，有彩色底部条带、layer class、status class，视觉上过于复杂。

### 模式 5: 顶部导航 — 轻量、上下文相关

Flowith 画布顶部：
- Logo | **Flow 名称(可编辑)** | 分享 | 全屏 | 返回
- 右侧: 积分状态 | 创作台

名称可编辑让用户有"这个画布属于我"的归属感。

**对比溯证 Agent**: 顶部有"中心观点"大输入框（只读），占据了太多空间。

### 模式 6: 空状态设计

Flowith 新画布是**完全空白**的（只有浅灰背景），等待用户输入。没有预设节点、没有示例内容。

**对比溯证 Agent**: 画布预置了 demo 案例的所有节点，用户一进来就被信息淹没。

## 三、溯证 Agent 具体改进建议

### 改进 1: 实现左侧边栏功能（最高优先级）

当前问题: 四个按钮点击无反应。

建议实现:
- **画布** — 切换到画布视图（当前已有）
- **Agent** — 打开 Agent 配置面板（可浮动或替换右侧面板）
- **知识库** — 显示证据/知识库列表视图
- **设置** — 打开设置面板（主题、模型选择等）

最小实现: 点击后至少切换主内容区，显示对应的占位内容。

### 改进 2: 添加首页/仪表盘入口

不要直接进入画布。先让用户看到一个仪表盘：
- 品牌标题 + 副标题
- 中央输入框: "输入一个观点，开始语境化可核查分解..."
- 或选择已有案例
- 点击"开始分析"后才进入画布

### 改进 3: 将输入栏移至画布底部中央

参照 Flowith，画布的输入应该在底部：
- 默认: 简洁输入框 "在此输入问题或指令..."
- 选中节点后: 输入框变为针对该节点的操作（展开/质疑/改写）
- 发送后生成新节点

### 改进 4: 简化节点样式

- 去掉彩色底部条带，改用更 subtle 的状态指示
- 圆角白色卡片 + 浅阴影
- 文字内容优先
- 选中时高亮边框，而非整体变色

### 改进 5: 顶部导航轻量化

- 移除"中心观点"大输入框（移到设置或初始化页面）
- 顶部显示: Logo | 当前任务名称（可编辑）| 工具按钮
- 右侧: 状态指示 | 重置按钮

### 改进 6: 空画布状态

- 新画布应该是空的
- 用户输入第一个观点后，生成根节点
- 再逐步展开子节点
- 提供 demo 案例作为"快速体验"选项，但不是默认状态

## 四、实施优先级

| 优先级 | 改进项 | 影响 | 工作量 |
|-------|-------|------|--------|
| P0 | 左侧边栏功能化 | 高 | 中 |
| P1 | 首页仪表盘 | 高 | 中 |
| P1 | 底部输入栏 | 高 | 中 |
| P2 | 节点样式简化 | 中 | 低 |
| P2 | 顶部导航轻量化 | 中 | 低 |
| P3 | 空画布状态 | 中 | 中 |
