# 真探 Agent —— 交互设计提示词

> **项目定位**: AI驱动的谣言核查与事实追踪系统
> **设计气质**: 调查记者的严肃感 +  Variant 粗野主义的字体力量 + 编辑式设计的留白呼吸
> **核心目标**: 让用户感到"专业、可信、想继续用"

---

## 一、设计哲学

### 1.1 品牌人格
真探 Agent 不是一个"聊天机器人"，而是一位**数字调查记者**。它的设计应该传递：
- **严谨**: 每一条信息都经过交叉验证
- **透明**: 核查过程完全可见，不黑箱
- **力量**: 真相是有力量的，设计应该彰显这种力量
- **克制**: 不喧哗，不恐吓，用事实说话

### 1.2 视觉隐喻
- **报纸编辑部**: 信息层级清晰，标题醒目，正文易读
- **侦探白板**: Canvas 上的节点像线索板上的照片和便签
- **法庭证据链**: 报告页像呈堂证供，逻辑严密

### 1.3 设计参考
- **Variant.com**: 粗野主义排版，超大字号，黑白为主，内容为王
- **The New York Times**: 编辑式网格，信息密度适中，阅读舒适
- **Linear**: 深色模式下的专业感，动画精准克制
- **Perplexity**: 搜索结果的清晰分层，来源透明

## 二、色彩系统

### 2.1 主色调（黑白为基，克制用色）
```
--ink-black:       #0a0a0f    (主标题、关键信息)
--charcoal:        #1d1d1f    (正文、标签)
--steel:           #4a4a4f    (次要文字)
--silver:          #86868b    (辅助文字、时间戳)
--cloud:           #f5f5f7    (页面背景)
--paper:           #ffffff    (卡片背景)
--border-subtle:   #e5e5e5    (分割线)
--border-medium:   #d1d1d6    (强调分割线)
```

### 2.2 Agent 色彩（饱和度克制，去荧光化）
```
--agent-rumor:     #b45309    (谣言检测 - 暗琥珀，警示但不刺眼)
--agent-fact:      #2563eb    (事实核查 - 沉稳蓝)
--agent-source:    #7c3aed    (信源验证 - 深紫)
--agent-report:    #16a34a    (报告生成 - 暗绿)
```

### 2.3 可信度色彩
```
--credibility-high:     #15803d    (高可信度 - 深绿)
--credibility-good:     #16a34a    (可信)
--credibility-medium:   #d97706    (存疑 - 暗橙)
--credibility-low:      #ea580c    (低可信度 - 锈红)
--credibility-critical: #dc2626    (严重存疑 - 暗红)
```

### 2.4 色彩使用规则
- **60% 无彩色**: 白、灰、黑构成主体
- **30% 主色**: Agent 色彩用于区分角色
- **10% 强调色**: 仅用于警示和关键状态
- **禁止**: 荧光色、渐变背景、阴影发光效果

## 三、字体系统

### 3.1 字体栈
```css
font-family:
  "Noto Serif SC",          /* 中文衬线 - 标题、引言 */
  "Source Han Serif SC",     /* 思源宋体回退 */
  "PingFang SC",             /* 苹方回退 */
  Georgia,                   /* 英文衬线 */
  serif;

font-family:
  "Noto Sans SC",           /* 中文无衬线 - 正文、UI */
  "PingFang SC",
  "Microsoft YaHei",
  -apple-system,
  sans-serif;

font-family:
  "JetBrains Mono",         /* 等宽 - 代码、时间戳 */
  "SF Mono",
  monospace;
```

### 3.2 字号层级（参考 Variant 的极端对比）
```
Display:     48px / 1.1  / 700    (首页主标题 - "真探 Agent")
Headline:    32px / 1.2  / 700    (页面标题)
Title:       24px / 1.3  / 600    (卡片标题、Agent 名称)
Subtitle:    18px / 1.4  / 500    (副标题、角色标签)
Body:        15px / 1.6  / 400    (正文、描述)
Caption:     13px / 1.5  / 400    (辅助文字、时间戳)
Mono:        13px / 1.4  / 400    (等宽文字、技术信息)
```

### 3.3 字体使用规则
- **标题用衬线**: Noto Serif SC，传递严肃、权威感
- **正文用无衬线**: Noto Sans SC，保证屏幕阅读舒适
- **技术信息用等宽**: JetBrains Mono，区分技术元数据
- **行高**: 标题 1.1-1.3，正文 1.6-1.8
- **字重**: 正文不超过 400，标题可用 600-700

## 四、布局原则

### 4.1 网格系统
```
页面最大宽度: 1200px
内容区域:      900px (居中)
侧边栏:        240px
边距:          24px (桌面) / 16px (移动)
```

### 4.2 留白规则
- **模块间距**: 48px (大模块之间)
- **卡片间距**: 24px
- **内容内边距**: 24px (卡片) / 16px (紧凑卡片)
- **元素间距**: 12px-16px
- **原则**: 宁多勿少，呼吸感优先于信息密度

### 4.3 卡片设计
```
border-radius: 12px (柔和但不过分)
border: 1px solid var(--border-subtle)
background: var(--paper)
box-shadow: none (扁平化，拒绝浮层感)
hover: border-color 变深，微上移 1px
```

## 六、动画原则

### 6.1 动画哲学
- **克制**: 动画是调味品，不是主菜
- **功能优先**: 每个动画都应有目的（引导注意力、反馈状态、解释变化）
- **物理感**: 缓动函数符合物理直觉

### 6.2 缓动函数
```css
--ease-out: cubic-bezier(0.16, 1, 0.3, 1);      /* 进入 */
--ease-in-out: cubic-bezier(0.65, 0, 0.35, 1);  /* 切换 */
--ease-spring: cubic-bezier(0.34, 1.56, 0.64, 1); /* 弹性（仅用于脉冲） */
```

### 6.3 具体动画

*页面切换*:
```
退出: opacity 1→0, translateY 0→-8px, 200ms, ease-in
进入: opacity 0→1, translateY 12px→0, 300ms, ease-out
```

*Agent 切换*:
```
旧 Agent: opacity 1→0, scale 1→0.95, 150ms
新 Agent: opacity 0→1, scale 1.02→1, 250ms (delay 100ms)
```

*步骤完成*:
```
节点: scale 0→1, 300ms, spring
对勾: stroke-dashoffset 动画, 200ms
连线: width 0→100%, 400ms, ease-out
```

*实时输出*:
```
新项: opacity 0→1, translateY 8px→0, 200ms
列表上移: translateY 0→-24px, 200ms
```

*进度条*:
```
width 变化: 300ms, ease-out
颜色过渡: 200ms
```

### 6.4 禁止的动画
- 旋转 loading（用脉冲代替）
- 弹跳、晃动等干扰性动画
- 超过 500ms 的长动画
- 同时多个元素动画（错峰 50ms）

## 七、交互细节

### 7.1 悬停状态
- 按钮: 背景色加深 5%，border 变深
- 卡片: border 变深，微上移 1px（translateY -1px）
- 链接: 下划线出现
- 节点: 放大 1.05 倍，显示 tooltip

### 7.2 点击反馈
- 按钮: scale 0.98，100ms
- 卡片: 背景微变
- 节点: 边框加粗，选中态

### 7.3 加载状态
- 骨架屏: 灰色块，呼吸动画（opacity 0.4→0.8，1.5s 循环）
- 进度指示: 明确进度条，不要无限 loading
- Agent 运行: 脉冲动画 + 实时输出

### 7.4 错误状态
- 背景: 浅红色（#fef2f2）
- 边框: 红色
- 图标: 警告三角形
- 文字: 明确的错误说明 + 重试按钮

## 九、可访问性

### 9.1 色彩对比
- 文字与背景对比度 >= 4.5:1
- 大文字 >= 3:1
- 不仅依赖颜色传达信息（配合图标、文字）

### 9.2 键盘导航
- Tab 顺序合理
- 焦点状态明显（2px outline）
- Enter/Space 激活按钮

### 9.3 屏幕阅读器
- 语义化 HTML 标签
- ARIA 标签（aria-label, aria-live）
- 状态变化通知（aria-live="polite"）

## 十、特别设计元素

### 10.1 真相印章
当核查完成时，结论卡片上出现一个"真相印章"动画：
- 圆形印章图案，从 scale 3→1，opacity 0→1
- 配合轻微的纸张纹理背景
- 印章文字: "已核查" 或 "VERIFIED"

### 10.2 证据链视觉
报告页右侧的证据链：
- 像侦探线索板一样
- 节点用图钉效果（小圆点）
- 边用红线连接
- hover 时高亮整条路径

### 10.3 打字机效果
Agent 实时输出的文本：
- 逐字出现的效果（可选，速度要快）
- 光标闪烁
- 完成后光标消失

### 10.4 时间戳
所有时间显示使用等宽字体：
- "2024.01.15 14:32"
- 灰色，小字号
- 精确到秒（用于 Agent 执行记录）
