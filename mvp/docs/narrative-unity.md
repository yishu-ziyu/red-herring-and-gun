## 一、一句话 Pitch

> **搜索引擎帮你找信息，我们帮你看清一个观点的结构。**

---

## 二、产品定位：Argument-Checking，不是 Fact-Checking

| 维度 | 传统 Fact-Checker | 红鲱鱼与枪 |
|------|------------------|-----------|
| 处理对象 | 单一事实陈述 | 含混观点 / 复合论证 |
| 输出 | "对"或"错" | 论证结构拆解 + 证据许可边界 |
| 核心问题 | "这句话是否为真" | "这句话里的每个判断分别需要什么证据、现有材料能支持到什么程度" |
| 典型用户场景 | 核实一条新闻 | 写论文/研报/政策分析前，确认论证能否站住 |

**底线**：我们不替用户下结论。我们帮用户看清一个论证的结构，然后让用户自己判断。

## 三、目标用户

### 黑客松 Demo 优先用户（当前阶段）

看到热点传言、截图、链接后需要快速判断"能不能转发"的人。

### 长期扩展用户

本科生、研究生、记者、内容创作者、政策研究者——任何需要"写一个有依据的观点"的人。

**讲述规则**：Demo 面向"热点核查"场景；提到学术/政策时，表述为"未来支持"，不要暗示当前版本已覆盖。

## 四、技术卖点

### 4.1 360 双向搜证

核心机制：对同一命题并行下发两条认知方向相反的检索指令。

- Direction A（支持性检索）：`Claim + "证据 来源 官方说明 原始出处"`
- Direction B（反驳性检索）：`Claim + "辟谣 反例 争议 无法证实"`

信源经过四维度审计（相关性、可追溯性、方法适配性、语境适配性），低信用线索在 Grader 节点被降级或剔除，绝不进入核心主证据链。

### 4.2 多 Agent 流水线

用户输入材料后，系统自动执行：

立案 → 原子命题拆解 → 多引擎并行检索 → 来源去重与独立性判断 → 交叉共识分析 → 证据许可判定 → 报告输出与闭环

每个 Agent 有明确的身份、工具、记忆边界和交接规则；过程在 Mission Control 中可见，不做黑箱。

### 4.3 证据边界报告

报告的核心不是"真假判定"，而是：

- **可以说**：哪些推论被现有证据许可
- **不能说**：哪些逻辑链存在证据缺口，强行推出属于越界
- **改写建议**：将越界表达降强度到证据允许的范围

证据分级（A-E）：

| 等级 | 定义 |
|------|------|
| A | 原始权威来源（政府官网、学术期刊、国际组织） |
| B | 专业二手分析 |
| C | 解释性材料 |
| D | 个案与舆论线索 |
| E | 不可用或高度可疑 |

## 五、诚实性表述：我们如何处理失败和不可用

以下规则适用于所有对外输出（路演、文档、界面文案）：

1. **搜索无结果 ≠ 没有反证**。表述为"暂未发现反证（搜索范围：XX）"，不说"没有反证"。
2. **来源转载 ≠ 独立验证**。同一篇文章被多个搜索引擎收录，只算 1 个来源。
3. **模型调用失败**：显示错误状态，不生成模拟分支或伪进度。
4. **证据不足的命题**：标记为"存疑"或"需人工复核"，不做推断。
5. **结论只描述证据允许的范围**：不说"事实为真"，说"当前证据链支持该结论，暂未发现反证"。
6. **降级策略公开**：某搜索引擎不可用时，UI 中标注"交叉验证受限"，不假装完整。

**核心理念**：诚实比看起来厉害更重要。

## 六、禁止使用的措辞

| 禁止 | 原因 | 建议替代 |
|------|------|---------|
| "完美" | 无法验证，且产品刻意保留不确定性 | "可审查的""透明的" |
| "终极武器" | 过度承诺，与诚实性原则矛盾 | "帮你思考的工具""清醒思考的伙伴" |
| "100% 准确" | 任何核查系统都无法保证 | "证据链完整度：……" |
| "事实为真" | 越界表述 | "当前证据链支持该结论，暂未发现反证" |
| "没有反证" | 搜索范围有限 | "暂未发现反证（搜索范围：……）" |
| "多源一致所以为真" | 混淆"转载同源"与"独立验证" | "多源独立验证"或"来源互相印证" |
| "来源可靠" | 模糊，未说明可靠标准 | "来源可追溯至原始出处" |
| "高置信度" | 无量化依据 | "证据链完整度：原始来源可追溯 + 独立来源互相印证 + 反证检索已完成" |
| "AI 会/不会取代……" | 产品不对命题本身下判断 | （让证据说话） |
| "一键查出真相" | 暗示确定性，忽视证据边界 | "帮你看清论证结构，自己判断" |
| "解决信息过载" | 不是信息聚合工具 | "帮你在信息中找到论证的落脚点" |
| "秒级出结果" | 实际依赖多轮检索 + LLM | "并行检索，30 秒内完成拆解" |

## 七、与竞品的差异声明（评委高频问题预判）

**Q：和 Perplexity / Consensus / Deep Research 有什么区别？**

> 它们以信息为中心——你问问题，它搜索并总结。  
> 我们以论证结构和证据许可为中心——你抛出一个观点，我们帮你拆解它的逻辑结构，告诉你每一步需要什么证据、现有材料能支持到什么程度。  
> 信息检索是输入，论证核查是输出。

**Q：如果用户不懂逻辑学，能看懂吗？**

> 节点用语都是自然语言——"能不能说导致""缺少什么证据"。不需要懂逻辑学，只需要有"写东西时被质疑过引用"的经历。

**Q：这会不会让表达变得过于保守？**

> 恰恰相反。如果证据足够强，我们会告诉你"可以保留原表述"。目标是准确，不是保守。
