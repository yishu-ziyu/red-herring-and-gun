# Lessons

## 2026-06-14: 记忆库只保存会改变下一次核查动作的内容

- 记忆库底座应走“案件即记忆”：完整案件可保存，但长期可召回的记忆索引必须由 Agent 判断生成。
- 不保存普通过程碎片、一次性域名记录和无行动价值的候选；只沉淀能影响下一次核查路径的经验，例如谣言套路、有效证据路径、转载链识别、关键判断边界、优先/避免的搜索策略。
- 记忆库 UI 不应让用户疲劳审核海量候选，而应解释“这次学到了什么、下次会怎么用、是否写入长期记忆”。

## 2026-05-31: UI 组件必须绑定真实产品状态

- 后续 UI / UX 组件必须承接真实功能、后台状态或核查判断价值；左侧 Kimi-like 区域只能表现中控 Agent 的真实过程流，不能做静态路径卡片或装饰性废话模块。

## 2026-05-29: 权限确认与执行边界

- 用户明确不希望安全的本地开发动作反复确认。
- 后续在 workspace 内的可逆读写、类型检查、构建、浏览器本地验收应直接执行。
- 仍需确认的边界：沙箱强制审批、破坏性命令、凭证/密钥、外部生产环境、不可逆数据变更。
